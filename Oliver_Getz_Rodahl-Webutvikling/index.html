Hi, and thank you for checking out my demo website. This is my "sandbox", so to speak, where I experiment with HTML/CSS to figure out how it all works together and get a sense of what kind of problems one might encounter when doing web development. Here are a few things I've learned recently:

  • Inline-blocks take HTML white spaces into account. Unless removed, these spaces will create gaps between elements.
  • Tags very much matter to search engines, i.e. good HTML is vital for Search Engine Optimization. Good practice is to understand what kind of content is in a block and use the appropriate tags. Proper tags also make it easier to target the necessary elements of a page in CSS without accidendally changing the style of wrong elements.
  • @media in CSS is great for responsive design! We wouldn't want this website to break when adjusting the window size, now would we?
  • Classes allow styling of multiple elements. How effective!
  • PHP outputs HTML/CSS/JavaScript based on code written by the developer and can grab info from a database like MySQL. Once the PHP code is processed, it is sent back to Apache then to the browser of the user as HTML/CSS/JavaScript. The PHP signal flow looks something like: Browser->Apache->PHP->MySQL->Apache->Browser.
